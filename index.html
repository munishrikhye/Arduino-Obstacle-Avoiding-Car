Introduction

Self-Driving Obstacle Avoiding Arduino Car that autonomously detects and avoids obstacles using an ultrasonic sensor. The green LED indicates a clear path, while the red LED and buzzer provide obstacle warning. The car uses DC motors with L298N motor drivers and servo motors for sensor rotation. The chassis is laser printed for sustainability, demonstrating energy-efficient operation and automation.

Objective

  • Build a self-driving obstacle-avoiding car using Arduino UNO.
  • Use LEDs and buzzer to give visual and audio feedback.
  • Automate sensor scanning with servo motors.
  • Ensure energy-efficient and reusable design.

Methodology

  1. Component Selection: Arduino, sensors, motors, LEDs, buzzer, servo motors, battery, chassis.
  2. Circuit Assembly: Connect components on breadboard and motor drivers, servo mounts for sensors.
  3. Programming: Code distance detection logic and LED/buzzer feedback.
  4. Testing & Optimization: Verify obstacle detection, motor control, and sensor rotation.
  5. Environmental Considerations: Reusable components and eco-friendly chassis.

Circuit Diagram

Environmental Impact Assessment

  • Materials: The Arduino Uno is RoHS and WEEE compliant, limiting hazardous substances.
  • Reusability: All components are modular and reusable, reducing electronic waste.
  • Packaging & Shipping: Ordering all components from a single supplier helps minimize carbon footprint.

Arduino Code

The following Arduino code controls the system, lighting LEDs and activating the buzzer based on distance readings from the HC-SR04 sensor.

 
  
// Define motor control pins
// ENA and ENB are PWM pins to control motor speed
// IN1, IN2 control the first motor direction
// IN3, IN4 control the second motor direction
// Pins assignment:
// 10 = ENA (Motor A speed control)
// 6 = IN1 (Motor A direction)
// 7 = IN2 (Motor A direction)
// 5 = IN3 (Motor B direction)
// 4 = IN4 (Motor B direction)
// 3 = ENB (Motor B speed control)

#include  // Include the Servo library to control a servo motor

Servo myServo; // Create a Servo object named myServo
long duration; // Variable to store the time taken for ultrasonic pulse to return (in microseconds)
int distance; // Variable to store the calculated distance from ultrasonic sensor (in cm)

// Define ultrasonic sensor pins
#define TRIG 11 // TRIG pin: sends the ultrasonic pulse
#define ECHO 8 // ECHO pin: receives the reflected pulse

void setup() {
 // Initialize serial communication at 9600 baud for debugging
 Serial.begin(9600);

 // Set ultrasonic sensor pins
 pinMode(TRIG, OUTPUT); // TRIG will send pulses
 pinMode(ECHO, INPUT); // ECHO will read pulses

 // Set motor pins as OUTPUT
 pinMode(10, OUTPUT); // ENA controls speed of Motor A
 pinMode(6, OUTPUT); // IN1 controls direction of Motor A
 pinMode(7, OUTPUT); // IN2 controls direction of Motor A
 pinMode(3, OUTPUT); // ENB controls speed of Motor B
 pinMode(5, OUTPUT); // IN3 controls direction of Motor B
 pinMode(4, OUTPUT); // IN4 controls direction of Motor B

 // Set both motors to move forward initially
 digitalWrite(6, HIGH); // Motor A: IN1 HIGH
 digitalWrite(7, LOW); // Motor A: IN2 LOW
 digitalWrite(5, HIGH); // Motor B: IN3 HIGH
 digitalWrite(4, LOW); // Motor B: IN4 LOW

 // Turn on motors at full speed
 digitalWrite(10, HIGH); // ENA HIGH → Motor A ON
 digitalWrite(3, HIGH); // ENB HIGH → Motor B ON

 // Print a startup message to the serial monitor
 Serial.println("Robot started");
}

void loop() {
 myServo.attach(9); // Attach servo to pin 9 (required to control it)

// Sweep servo from 0° to 120° in steps of 15°
for (int angle = 0; angle < 130; angle += 15) {
 myServo.write(angle); // Move servo to current angle

 // --- Ultrasonic distance measurement ---
 digitalWrite(TRIG, LOW); // Make sure TRIG is LOW before sending pulse
 delayMicroseconds(2); // Short delay to stabilize
 digitalWrite(TRIG, HIGH); // Send ultrasonic pulse
 delayMicroseconds(10); // Keep pulse HIGH for 10 microseconds
 digitalWrite(TRIG, LOW); // Stop the pulse

 duration = pulseIn(ECHO, HIGH); // Read the time for the echo to return

 distance = duration * 0.034 / 2; // Convert duration to distance in cm (speed of sound formula)

   // Print distance to serial monitor
   Serial.print("Distance: ");
   Serial.print(distance);
   Serial.println(" cm");

   // Check if an object is closer than 10 cm
   if (distance > 0 && distance <= 10) {
    // Stop both motors if obstacle detected
    digitalWrite(10, LOW); // Motor A OFF
    digitalWrite(3, LOW); // Motor B OFF
    Serial.println("Obstacle detected Motors STOP");
   } else {
    // Otherwise, keep motors moving forward
    digitalWrite(6, HIGH); // Motor A IN1 HIGH
    digitalWrite(7, LOW); // Motor A IN2 LOW
    digitalWrite(5, HIGH); // Motor B IN3 HIGH
    digitalWrite(4, LOW); // Motor B IN4 LOW

    digitalWrite(10, HIGH); // Motor A ON
    digitalWrite(3, HIGH); // Motor B ON
   }

   delay(500); // Wait 0.5 seconds before next loop iteration
  }

  myServo.write(0); // Return servo to starting position (0°) after sweep
}

Conclusion

The system shows a car that can detect obstacles using an ultrasonic sensor and a servo motor, and it stops when something is in the way. We ran into a bunch of issues while building it. One problem was that the motors weren’t getting enough current. The 9-volt battery didn’t give enough power, so in the future we should use lithium-ion batteries. For now, we’re using the lab power supply. Another issue was the wiring. Since we used prototype wires, they were unstable and got loose easily. A good solution for the future is to solder all the connections so everything stays secure.
